## Honest Evaluation for YOUR Use Case

### 🏆 Current: Camoufox (What We Use Now)

### 1. `browser-use` — LLM-Controlled Browser

**Verdict: ❌ WRONG TOOL for this project**

| Factor | Assessment |
|--------|-----------|
| **Intelligence** | Uses LLM to figure out where to click, what to type |
| **Our reality** | We ALREADY know the selectors, the URLs, the page structure |
| **Cost** | Every browser action = LLM API call = $$$ per run |
| **Speed** | LLM reasoning loop adds 5-15 seconds PER action |
| **Reliability** | LLM can hallucinate clicks, misidentify elements |

**Why it's tempting but wrong:** It looks cool because it "thinks" — but we don't need thinking. We need a browser to load a URL and hand us the HTML. Our DOM parser already knows exactly where every data point lives. Adding an LLM layer between "go to URL" and "get HTML" is like hiring a PhD to press a light switch.

**WHERE it WOULD be useful:** Your OTHER projects — like scraping NSE/BSE corporate filings, SEBI enforcement orders, NCLT petitions, government portals. Those sites have complex navigation, login flows, CAPTCHAs, and unpredictable layouts. `browser-use` shines there.

---

### 2. `Obscura` — Rust Headless Browser

**Verdict: ⚡ VERY INTERESTING, but wait**

| Factor | Assessment |
|--------|-----------|
| **Memory** | ~30MB vs Camoufox's ~300-500MB — **10x lighter** |
| **Speed** | Much faster startup |
| **Stealth** | Built-in anti-fingerprinting |
| **Compatibility** | Puppeteer + Playwright compatible |
| **Maturity** | **NEW** — this is the risk |

**The real math for your daemon:**

```
Current (Camoufox):
  1 browser instance running all day = ~400MB RAM
  60 requests/day = works fine

Obscura:
  1 browser instance = ~30MB RAM
  Could run 5 tickers simultaneously in same RAM as 1 Camoufox
```

**BUT the critical question:** Can Obscura bypass Cloudflare Turnstile on `perplexity.ai`? Camoufox is **proven** — it uses patched Firefox fingerprints specifically designed to defeat Cloudflare. Obscura is promising but untested against Perplexity's specific protection.

**My recommendation:** Keep Camoufox now. When Obscura matures (give it 3-6 months), test it against Perplexity. If it passes Cloudflare, switch — the memory savings are enormous for multi-ticker daemons.

---

### 3. `Scrapling` — Smart Scraping Framework

**Verdict: ✅ WORTH ADDING as a parsing layer**

| Factor | Assessment |
|--------|-----------|
| **Adaptive selectors** | If Perplexity changes their HTML layout, Scrapling can **relocate** elements automatically |
| **Cloudflare handling** | Built-in — could replace Camoufox for the browser layer |
| **AI/MCP integration** | Can feed data directly to LLMs |
| **Our pain point** | Our chunk-based parser (`|||` separator) is fragile — if Perplexity changes ONE div class, it breaks |

**This is the one that actually solves a real problem we have.** Our current parser in [finance_scraper.py](file:///home/vvd/prakash/perpexicity-finance/perplexity-finance-scraper/scraper/finance_scraper.py) uses:
- Regex patterns for dates, prices, changes
- Fixed section markers ("Stories & Analysis", "Key Issues", "Peers")
- Chunk-based text splitting with `|||`

If Perplexity renames "Key Issues" to "Hot Topics" or restructures their DOM, **our parser silently breaks**. Scrapling's adaptive selectors would handle this.
